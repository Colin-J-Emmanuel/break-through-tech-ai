# Unit 9 — Responsible ML

The final unit of the Machine Learning Foundations curriculum shifts from *building* models to
*preparing them for the real world*. It is a **theory and reflection unit** — there are no
graded notebooks, so this folder documents the concepts rather than hosting code.

The unit asks a harder question than "does the model work?": **should it be deployed, and for
whom does it work?**

## Topics covered

| Topic | What it addresses |
|---|---|
| **Solution Engineering** | Framing an ML solution around a real business or user need, and knowing when ML is (and isn't) the right tool. |
| **Data Sufficiency** | Whether the available data is enough — in quantity, quality, and representativeness — to support the intended model and claims. |
| **Feature Issues** | Problems that hide in features: leakage, proxies for sensitive attributes, instability, and bias baked into inputs. |
| **Fairness & Accountability** | Metrics and *types* of fairness, how to measure disparate performance across groups, and how to build accountability into an ML system. |

## Why it matters

Everything in Units 1–8 optimizes for predictive performance. Unit 9 reframes that: a model
with excellent accuracy can still be unfair, brittle, or inappropriate for its context. This
unit connects directly to the AI Studio work, where models were built for a real industry
partner and had to be reliable and interpretable — not just accurate on a held-out set.
